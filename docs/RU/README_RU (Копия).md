Модульная медиасистема для обработки, управления и резервного копирования, предназначенная для:
- рабочего процесса создания медиаконтента
- настройки и оптимизации системы
- операций резервного копирования и восстановления
- автоматизации производственного процесса

## Особенности архитектуры дисковых разделов

Перед установкой **REINCARNATION BACKUP KIT (backupkit)** необходимо создать на дисках компьютера файловый раздел, смонтированный как:
- `/mnt/backups`, в котором **REINCARNATION BACKUP KIT (backupkit)** будет создавать архивы настроек Firefox, пользовательских данных и системных данных. При запуске **PRODUCTION MEDIA PANEL (mediapanel)** этот дисковый раздел будет хранить архивы видеопроектов, а также содержать корзину для удаления ненужных видеофайлов.


Для установки **REINCARNATION MEDIA (remedia)** используйте каталог `/mnt/shotcut` для медиафайлов Shotcut, что позволит Shotcut работать быстрее. Этот раздел создается на быстром SSD-накопителе. Здесь устанавливается Ubuntu, и создается основной каталог пользователя.

Сразу после установки **REINCARNATION MEDIA (remedia)** используйте скрипт `hdd-setup-profiles.sh` для создания раздела `/mnt/storage`, который будет расширением каталога для первого пользователя видеосистемы и будет содержать каталоги для хранения больших медиафайлов (`Videos`, `Images` и `Music`). Вы также можете создать каталоги для двух других пользователей компьютера и операционной системы. После создания системных разделов вы можете запустить скрипт setup-symlinks.sh для создания символических ссылок на каталоги /mnt/storage/Videos, /mnt/storage/Images и /mnt/storage/Music, чтобы эти каталоги отображались в пути /home/user/... учетной записи пользователя.

При установке **REINCARNATION MEDIA (remedia)** вы можете выбрать установку только **REINCARNATION BACKUP KIT (backupkit)**.

См. также:
- Разметка SSD + HDD для Linux (для монтирования в Shotcut) [docs/EN/README_SSD_SETUP_EN.md](docs/EN/README_SSD_SETUP_EN.md)

## 🧩 Обзор архитектуры REINCARNATION MEDIA (remedia)

REMEDIA разделена на четыре независимых, но совместимых комплекта:

### 1. 🎬 REINCARNATION MEDIA (remedia) Core
Инструменты для настройки и оптимизации среды на системном уровне.

Включает:
- Инспектор монтирования системы
- Профили конфигурации жестких дисков/хранилищ
- Менеджер символических ссылок
- Инструмент установки/удаления CUDA TOOLS
- Просмотрщик логов
- Интерфейс переключения языков
- Скрипты установки/удаления
Ключевые скрипты:
- `check-cuda-tools.sh`
- `hdd-setup-profiles.sh`
- `setup-symlinks.sh`
- `show-system-mounts.sh`

🚀 Установка
1. Клонируйте этот репозиторий:
```bash
git clone https://github.com/krashevski/remedia.git
cd remedia
```

2. Разработка и тестирование
`run.sh` используется только для тестирования и отладки:
```bash
./run.sh
```

3. Запустите установку:
```bash
sudo ./install.sh
```

Что устанавливается:
- remedia (ядро CLI)
- mediasystem (процессор)
- mediapanel (панель управления с графическим интерфейсом)
- backupkit (инструменты резервного копирования и восстановления)

4. Проверка системы:
```bash
remedia-doctor
```

5. Команды:
- `remedia`
- `mediasystem`
- `mediapanel`
- `backupkit`

6. Компоненты:
Каждый компонент модульный:
- основная логика → /lib/remedia/
- CLI-оболочка → /usr/local/bin/
- configs → /etc/remedia/

### 2. 🛠 MEDIA SYSTEM (mediasystem)
Уровень конфигурации среды ОС для графических процессоров и Nvidia, а также установщик приложений для обработки мультимедиа
Назначение:
- Управление зависимостями системы
- Унификация пользовательского интерфейса терминала (интеграция Gnome с терминалом)
- Установка и настройка программного обеспечения для обработки мультимедиа
Ключевые компоненты:
- `mediasystem.sh`
- `modules/`
- `packages/`
- Уровень интеграции с терминалом
> [I] `packages/` В каталоге находятся простые файлы со списком программ для установки из репозиториев, которые можно редактировать.
Запустите `remedia` после установки:
```bash
mediasystem
```
См. также:
- MEDIA SYSTEM (mediasystem) [docs/EN/README_MEDIASYSTEM_EN.md](docs/EN/README_MEDIASYSTEM_EN.md)

### 3. 🎬 PRODUCTION MEDIA PANEL (mediapanel)
Основной интерфейс для рабочего процесса производства.

Назначение:
- Управление медиафайлами на основе проектов
- Конвейер обработки видео/аудио
- Генерация прокси
- Разделение сцен
- Инструменты синхронизации
- Система экспорта и архивирования
Особенности:
- Конвейер обработки данных телефона
- Система управления жизненным циклом проекта
- Система блокировки на основе PID
- Отслеживание статуса проекта
- Система логирования (история производства)
- Интеграция с Shotcut
Запуск после установки `remedia`:
```bash
mediapanel
```
См. также:
- **MEDIA SYSTEM (mediasystem)** [docs/EN/README_MEDIAPANEL_EN.md](docs/EN/README_MEDIAPANEL_EN.mdd)

### 4. 💾 REINCARNATION BACKUP KIT (backupkit)
Полный набор инструментов для резервного копирования и восстановления системных и пользовательских данных, а также Firefox. Особенности:
- Резервное копирование профиля Firefox
- Резервное копирование и восстановление пользовательских данных
- Резервное копирование и восстановление данных той же версии
- Автоматизация Cron
- Утилита очистки журналов резервного копирования
- Последняя версия программы для просмотра резервных копий
Поддерживаемые операционные системы:
- Debian 12
- Ubuntu 22.04 / 24.04 / 25.10
Ключевые скрипты:
- `backup-system.sh`
- `backup-userdata.sh`
- `restore-system.sh`
- `restore-userdata.sh`
- `backup-restore-firefox.sh`
- `add-cron-backup.sh`
- `remove-cron-backup.sh`
Запуск:
```bash
backupkit
```

> [!] При установке новой версии операционной системы восстановление системных данных не требуется и может быть опасным. Восстанавливайте только пользовательские данные, данные Firefox и настройки учетной записи пользователя.

См. также
- **КОМПЛЕКТ РЕЗЕРВНОГО КОПИРОВАНИЯ PREINCARNATION (backupkit)** [docs/EN/README_BACKUPKIT_EN.md](docs/EN/README_BACKUPKIT_EN.md)

## 🔧 Требования

- bash 5+
- rsync
- systemd (необязательно)

## 📦 Архитектура REINCARNATION MEDIA (remedia)

/usr/local/bin/ → Команды CLI
/usr/local/lib/remedia/ → ядро ​​системы
/usr/local/lib/<module>/ → компоненты
/var/log/remedia/ → журналы
/etc/remedia/ → конфигурация

## ⚠️ Примечания по стабильности REINCARNATION MEDIA (remedia)

Эта система находится в активной разработке и должна использоваться при следующих условиях:
- включено корректное резервное копирование
- предварительно протестированная среда
- стабильное монтирование дисков (рекомендуется структура /mnt)
Критические механизмы безопасности:
- блокировка проекта на основе PID
- автоматическая очистка устаревших блокировок
- неразрушающие рабочие процессы (корзина) система)
- разделенные уровни хранения
