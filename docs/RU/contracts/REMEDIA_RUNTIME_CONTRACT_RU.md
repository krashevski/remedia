# 🔧 КОНТРАКТ СРЕДЫ ВЫПОЛНЕНИЯ REMEDIA (СПЕЦИФИКАЦИЯ В СТИЛЕ BASH SYSTEMD)

КОНТРАКТ СРЕДЫ ВЫПОЛНЕНИЯ REMEDIA v1
--------------------------------------------------
Этот файл определяет гарантии выполнения, среду,
и правила жизненного цикла для ВСЕХ компонентов REMEDIA
-------------------------------------------------

## 🧠 1. ОСНОВНОЙ ПРИНЦИП (ИНВАРИАНТ СИСТЕМЫ)

- Ядро неизменяемо
- Среда определяет среду
- Модули являются исполнителями без состояния
- Ни один компонент не может жестко задавать пути к файловой системе

## ⚙️ 2. КОНТРАКТ СРЕДЫ (ПЕРЕНОСНЫЕ ПЕРЕМЕННЫЕ)

### ОБЯЗАТЕЛЬНЫЕ ПЕРЕМЕННЫЕ
- РЕЖИМ # system
- РЕЖИМ УСТАНОВКИ # deb 
- ПРЕФИКС # разрешенный корневой префикс
- REMEDIA_LIB
- REMEDIA_BIN
- REMEDIA_VAR
- REMEDIA_LOG

### ПРАВИЛА
- РЕЖИМ определяется (никогда не доверяется вручную)
- ПРЕФИКС определяется из РЕЖИМА
- REMEDIA_* определяется из ПРЕФИКСА
- модули НЕ ДОЛЖНЫ изменять ENV

## 📦 3. КОНТРАКТ ФАЙЛОВОЙ СИСТЕМЫ

### РЕЖИМЫ СИСТЕМЫ

```bash
/usr/bin/remedia
/usr/lib/remedia
/etc/remedia
/var/log/remedia
```

## 🧱 4. КОНТРАКТ ЯДРА

ЯДРО ДОЛЖНО:
- находиться в /usr/lib/remedia/core
- никогда не зависеть от INSTALL_MODE
- никогда не выполнять операции ввода-вывода вне путей среды выполнения
- никогда не использовать sudo
- никогда не предполагать структуру файловой системы

## ⚙️ 5. КОНТРАКТ СРЕДЫ ВЫПОЛНЕНИЯ

СРЕДСТВО ВЫПОЛНЕНИЯ ДОЛЖНО:
- определять РЕЖИМ
- определять ПРЕФИКС
- генерировать пути REMEDIA_*
- гарантировать существование каталога
- быть ЕДИНСТВЕННЫМ местом, где определены пути

> [!] ✔ ПРАВИЛО
> [!] Только runtime_init() может создавать состояние файловой системы.

## 🧩 6. КОНТРАКТ МОДУЛЯ

МОДУЛИ ДОЛЖНЫ:
- использовать ТОЛЬКО переменные REMEDIA_*
- не запускать /usr/lib напрямую
- не вычислять пути
- не определять окружение
- не вызывать sudo

## 🔥 8. КОНТРАКТ ВЫПОЛНЕНИЯ (ПОТОК ВРЕМЕНИ ВЫПОЛНЕНИЯ)

```text
ENTRY POINT (/usr/bin/remedia)
↓
core/bootstrap
↓
runtime_init()
↓
resolve_mode()
↓
init_paths()
↓
cli_parse()
↓
run_pipeline()
↓
module execution
```

## 🧠 9. КОНТРАКТ ИСПОЛНИТЕЛЯ

safe_run():
- никогда не приводит к сбою системы
- регистрирует ошибки
- возвращает только код ошибки

run_as_root():
- только повышается на границе выполнения
- никогда не используется внутри модулей

## ⚠️ 10. ОШИБКА КОНТРАКТА

Система НИКОГДА НЕ ДОЛЖНА:
- записать в /usr во время выполнения
- выполнить модуль без runtime_init

## 🔒 11. ПРАВИЛА НЕИЗМЕННОСТИ

- CORE доступно только для чтения после установки
- состояние времени выполнения эфемерно
- модули сменные

## 🧠 12. МЕНТАЛЬНАЯ МОДЕЛЬ (КЛЮЧЕВОЙ)

* CORE = двигатель (никогда не меняется)
* ВРЕМЯ ВЫПОЛНЕНИЯ = мозг (решает окружающая среда)
* МОДУЛИ = мышцы (выполнение задач)
* FILESYS = результат решения во время выполнения
