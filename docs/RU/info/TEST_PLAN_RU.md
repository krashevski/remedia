# REMEDIA — STABILITY TEST PLAN

## 1. Цель

Обеспечить стабильную работу системы REMEDIA:

* Установка проходит без ошибок
* Компоненты работают независимо
* Удаление не ломает систему
* Система восстанавливается после сбоев
* Нет остаточных файлов после удаления

## 2. Компоненты

* remedia (core)
* mediapanel (GUI launcher)
* mediasystem (runtime)
* backupkit (backup / recovery)

## 3. Окружение

* OS: Linux (Ubuntu / GNOME)
* Права: sudo required
* Пути:
- `/usr/local/bin/`
- `/usr/local/lib/`
- `/usr/share/applications/`
- `/etc/`

## 4. Типы тестирования

## 5. Формат тест-кейса

```text
TEST ID: XXXX
Название:
Шаги:

```

## 6. Риски

* Повреждённые пути
* Отсутствие прав
* Частичная установка
* Сломанные конфиги (`profiles.ini`)
* Конфликт версий
* Залипшие lock-файлы в `/etc/remedia`
* Блокировка установки/удаления
* Неконсистентное состояние системы

## 7. Критерии успешности

* Все тесты PASS
* Нет критических ошибок
* Система восстанавливается
* Нет “битых” состояний

## 8. Логирование

* Все тесты записываются в `logs/test.log`
* Формат:

[DATE] [TEST_ID] [STATUS] [MESSAGE]


## 9. Итоговый отчёт

* PASS / FAIL count
* Список ошибок
* Общий статус системы
