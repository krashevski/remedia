#  MEDIA SYSTEM (mediasystem)

[🇬🇧 English](../EN/README_MEDIASYSTEM_EN.md) | [🇷🇺 Русский](README_MEDIASYSTEM_RU.md)

## Обзор

Система **MEDIA SYSTEM (mediasystem)** для управления конвейерами установки программ обработки медиафайлов в средах Linux.

Основной скрипт конвейера — `mediasystem.sh` управляет всеми модулями в правильном порядке с корректным логированием. 

## Особенности

- Автоматическая установка необходимых пакетов (APT, Flatpak и т. д.)
- Модульный конвейер с независимыми этапами
- Простая настройка пользовательских каталогов программ мультимедиа
- Централизованная обработка логов с метками времени
- Безопасное удаление с помощью `uninstall_mediasystem.sh`

## Требования

- Система на базе Linux (рекомендуется Debian/Ubuntu)
- Bash 5.1+
- Основные утилиты: `grep`, `xargs`, `tar`
- Дополнительно: доступ `sudo` для установки пакетов

## Использование конвейера `mediasystem`

Конвейер выполнит следующие действия:
- Обновит репозитории APT и установит необходимые базовые системные и видео пакеты.
- Проверит использование GPU, уcтановит и настроит драйвер Nvidia, Cuda toolkit, проверит поддержку GPU пакетом ffmpeg, проверит работу Opengl.
- Установит приложения APT в соответствии с указаниями.
- Установит приложения Flatpak в соответствии с указаниями.
- Установит приложения SNAP в соответствии с указаниями.
- Проверит поддержку GPU программой Shotcut. 
- Сгенерирует сводный лог в `logs/`.

### Интерактивный выбор режима коныейера:
- безопасный — минимальные изменения, подходит для тестирования
- стандартный — установка модулей необходимых для видеоредактора Shotcut
- полный — установка всех программ Ubuntu/Linux для производства медиа в соответствии с указаниями

### Ведение журнала:
Весь вывод записывается в файл /var/log/remedia/mediasystem/pipeline.log. Проверьте этот файл, если какой-либо модуль не работает. Пример:
```bash
/var/log/remedia/mediasystem/
├── pipeline.log
├── 60_apt_packages.log
├── 10_snap_packages.log
└── 00_detect_gpu.log
```

### Особенность конвейера
При запуске `mediasystem.sh` пользователь может выбрать режим в интерактивном режиме или установить его в окружении:
```bash
export PIPELINE_MODE=full ./mediasystem.sh
```

### Удаление конвейера `mediasystem`

Для безопасного удаления установленных пакетов, приложений Flatpak и файлов конфигурации:
```bash
./uninstall_mediasystem.sh
```

- Автоматически удаляет пакеты APT, перечисленные в `packages/apt/`.
- Удаляет приложения Snap и Flatpak, если они присутствуют.
- Очищает файлы конфигурации для Shotcut и OBS Studio.
> [!] Скрипт удаления по умолчанию работает в безопасном режиме, поэтому ресурсоёмкие или необязательные модули могут быть пропущены.

## Ubuntu против Debian

| ОС | Примечания к первому запуску | Распространенные проблемы и способы их решения |

| ---------- | ------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------ |

| **Ubuntu** | - Обычно содержит актуальные пакеты. <br> - Запускайте с `sudo` при установке системных пакетов. | - В LTS-версиях могут отсутствовать некоторые старые зависимости. <br> - Убедитесь, что установлены `curl`, `wget`, `git` и `build-essential`. |

| **Debian** | - Стандартная стабильная версия может быть старше; может потребоваться включение `backports`. <br> - Используйте `sudo` для команд APT. | - Отсутствующие зависимости (например, `python3-venv`, `ffmpeg`) часто требуют ручной установки. <br> - Некоторые модули могут не работать в старых версиях библиотек. |

### Краткие советы по настройке для обеих систем:
```bash
# Сначала обновите систему
sudo apt update && sudo apt upgrade -y

# Установите необходимые инструменты
sudo apt install -y curl wget git build-essential python3-venv

# Необязательно: включите поддержку бэкпортов в Debian для новых пакетов
# echo "deb http://deb.debian.org/debian $(lsb_release -cs)-backports main" | sudo tee -a /etc/apt/sources.list
```

> 💡 Совет: Скрипт `/usr/local/bin/mediasystem` регистрирует ход выполнения и автоматически обрабатывает большинство ошибок. В Debian, если модуль не работает из-за отсутствующих библиотек, установите необходимые пакеты вручную и запустите скрипт снова.

## Настройка

- Пакеты и приложения APT: `packages/apt_utils.txt` `packages/apt_media.txt` (один пакет на строку, допускаются комментарии)
- Приложения Flatpak: `packages/flatpak.txt`
- Приложения SNAP: `packages/snap.txt`

## Устранение неполадок

- Если модуль не работает, проверьте соответствующий файл журнала в `$LOG_DIR`.
- Убедитесь, что `sudo` доступен для установки пакетов.
- Убедитесь, что Flatpak установлен и правильно настроен в вашей системе.
- Убедитесь онлайн, что пакет для версии ОС поддерживается APT или SNAP.
- В случае проблем с удалением убедитесь, что `PIPELINE_MODE` установлен в `safe` (скрипт удаления делает это автоматически).

## Вклад

- Добавьте модули в `modules/`
- Используйте стандартные функции логирования (`log_info`, `log_warn`, `log_error`)
- Отправляйте запросы на слияние для проверки
