# PRODUCTION MEDIA PANEL (mediapanel)

[🇬🇧 English](../EN/README_MEDIAPANEL_EN.md) | [🇷🇺 Русский](README_MEDIAPANEL_RU.md)

**PRODUCTION MEDIA PANEL (mediaoanel)** — это интерактивная панель Linux для управления видеопроектами, импорта видео с телефона, создания прокси-файлов, редактирования, экспорта и архивирования.

Разработана для документальных и творческих проектов.

## Ключевые особенности

1. **Состояние системы** — отображает GPU, NVENC, доступное пространство и количество проектов.
2. **Информация о проекте** – отображает:
- количество клипов в `footage`
- количество прокси-файлов
- размер папки
- общую статистику проекта, дату последнего изменения и этап жизненного цикла
3. **Создать проект** – создает проект в двух местах:
- Хранилище: `/mnt/storage/Videos/projects/ProjectName`
- Shotcut: `/home/user/shotcut/projects/ProjectName`
4. **Загрузка видео с телефона** – импорт видео с телефона:
- оригиналы в формате `raw`
- рабочие файлы в формате `footage`
5. **Создать прокси** – создает прокси-файлы в папке Shotcut с кодировкой UTF-8 и настройками безопасности.
6. **Запустить Shotcut** – открывает Shotcut (Flatpak).
7. **Экспорт на YouTube** – экспортирует готовое видео в папку `exports` с настройками YouTube.
8. **Архивировать проект** – сохраняет проект видео в архив `.tar.gz`.
9. **Инструменты для видео/графики/аудио** – быстрый доступ к приложениям APT, SNAP и Flatpak:
- Shotcut, OBS Studio
- GIMP, Krita
- Audacity

## Структура проекта

**Хранилище (оригиналы)**
/mnt/storage/Videos/projects/ProjectName
* footage/
* raw/
* edit/
* export/

**Shotcut (рабочее пространство)**
/home/vladislav/shotcut/projects/ProjectName
* proxy/
* project/

## 🔧 Требования PRODUCTION MEDIA PANEL (mediaoanel)

- Bash 5+
- rsync
- tar / gzip
- GNOME Terminal (рекомендуется)
- `/mnt/backups` смонтированный раздел файловой системы, на этом разделе диска будут сохраняться архивы проектов видео и будет корзина для последующего удаления видео мусора.
- `/mnt/shotcut` смонтированный раздел файловой системы для Shotcut прокси
- `/mnt/storage` смонтированный раздел файловой системы для каталогов хранения больших медиафайлов `Видео`, `Изображения`, `Музыка`

## Установка PRODUCTION MEDIA PANEL (mediaoanel)

Установка:
1. Клонируйте этот репозиторий:
```bash
git clone https://github.com/krashevski/remedia.git
cd remedia
```

2. Запустите установку:
```bash
sudo ./install.sh
```

3. Запуск:
```bash
remedia
```

4. В меню выберите: **Медиа cистема (установка и настройка программ)**  и выберите: **Полный (стандартный + все дополнительные модули)**.
5. В меню выберите: **Медиа панель (производство медиа)**
6. Убедитесь, что установлены медиа приложения APT и Flatpak:
- Shotcut
- OBS Studio
- GIMP
- Krita
- Audacity

## 🔐 Принципы проектирования PRODUCTION MEDIA PANEL (mediapanel)

✔ Модульность
Каждый комплект работает независимо:
- отсутствие жесткой зависимости между модулями
- совместное использование компонентов только через shared-li
✔ Отказоустойчивая архитектура
- set -euo pipefail используется в ядре скрипты
- Система блокировки на основе PID в инструментах для производства
- Автоматическая очистка устаревших блокировок
- Безопасное удаление медиафайлов через корзину
✔ Рабочий процесс, ориентированный на производство
Система разработана для:
- длительных проектов
- рабочих процессов с большим объемом медиаконтента (редактирование 4K)
- неразрушающей обработки
- поэтапных конвейеров (RAW → РЕДАКТИРОВАНИЕ → ЭКСПОРТ)'
✔ Журналы в удобочитаемом формате
Каждый проект поддерживает:
```bash
.log — временная шкала истории производства
.status — состояние проекта (НОВЫЙ / В ПРОЦЕССЕ / ГОТОВО)
.lock — защита активной сессии
```

## Примечания

Все операции безопасны для кодировки UTF-8 и пробелов в именах файлов.
Прокси создаются в отдельной рабочей папке `/mnt/shotcut` на SSD диске для ускорения редактирования.
Интеграция с Shotcut позволяет сразу использовать прокси для редактирования больших проектов 4K.
